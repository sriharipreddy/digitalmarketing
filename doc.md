### ‚úÖ USE CASE 1 ‚Äî "I need to generate leads for my business"

**Who:** Agencies, SaaS, coaches, consultants, B2B companies, service businesses

- **CRM Pipeline** ‚Äî Kanban view of all leads by stage (New ‚Üí Contacted ‚Üí Qualified ‚Üí Closed)
- **Multi-channel Traffic** ‚Äî Google Ads, Meta Ads, LinkedIn Ads, organic SEO all feeding into the same funnel
- **Automation** ‚Äî form submit ‚Üí instant email confirmation ‚Üí sales alert SMS ‚Üí 5-day nurture sequence
- **B2B Cold Outreach** ‚Äî LinkedIn + email sequences to ideal customer profiles

**Success Metrics:** Cost per lead (CPL), form conversion rate, lead-to-customer rate, pipeline value

### ‚úÖ USE CASE 2 ‚Äî "I want my website on Google page 1"

**What They Want:** Rank for high-intent keywords that bring free, organic traffic.

- **Schema Markup Generator** ‚Äî generates JSON-LD for Product, Article, FAQ, LocalBusiness, Review automatically
- **Backlink Builder** ‚Äî find link opportunities, send outreach emails, track gained/lost backlinks
- **Content Brief Generator** ‚Äî AI creates an SEO brief (word count, headings, keywords, FAQs) for every target keyword
- **Competitor Gap Analysis** ‚Äî see which keywords competitors rank for that you don't, one-click to create content
- **Google Search Console Integration** ‚Äî import real impressions, clicks, CTR, position data

**Success Metrics:** Keyword rankings, organic traffic, Domain Authority, backlinks gained, CTR improvement

## Database Design

All tables prefixed `mkt_` ‚Äî coexist safely with existing LicensedTaxi tables.
All primary keys: `CHAR(36) DEFAULT (UUID())` ‚Äî matching existing pattern.
All tables have `deleted_at DATETIME NULL` ‚Äî soft deletes everywhere.

### Core Tables by Service

**Every table has these standard columns (mirrors existing platform pattern):**
- `id CHAR(36) PRIMARY KEY DEFAULT (UUID())`
- `workspace_id CHAR(36) NOT NULL` (except global tables like `core_users`, `core_plans`)
- `created_at DATETIME DEFAULT CURRENT_TIMESTAMP`
- `updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP`

- `core_rate_limit_buckets` ‚Äî Redis-backed but mirrored here for analytics. Per-workspace + per-IP counters

### Per-Table Soft Delete Reconciliation with GDPR

Soft delete (`deleted_at`) is fine for accidental-deletion recovery, but it **conflicts with GDPR right-to-be-forgotten (Article 17)**.

**Resolution:**
- User-initiated delete ‚Üí sets `deleted_at` (soft delete, 30-day recovery window)
- DSAR erasure request ‚Üí triggers `mkt-rtbf-purge` queue ‚Üí **hard-delete from MySQL/PostgreSQL + ClickHouse + storage (S3 or local disk via `storage.deletePrefix()`) + Elasticsearch (when added)**, leaving only an anonymised audit-log entry with a hashed reference (legal retention requirement)
- See `compliance.md` for the full erasure pipeline

### ClickHouse Analytics Schema

```sql
-- High-volume time-series events (1,000+ events/sec)
CREATE TABLE analytics_events_ch (
  workspace_id  UUID,
  event_type    LowCardinality(String),  -- 'page_view','click','form_submit','purchase'
  session_id    String,
  user_id       Nullable(UUID),
  url           String,
  utm_source    Nullable(String),
  utm_medium    Nullable(String),
  utm_campaign  Nullable(String),
  country_code  LowCardinality(String),
  device_type   LowCardinality(String),
  revenue       Nullable(Decimal(12,2)),
  created_at    DateTime64(3)
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(created_at)
ORDER BY (workspace_id, created_at, event_type);

-- Real-time hourly aggregation (powers live dashboard)
CREATE MATERIALIZED VIEW analytics_events_hourly
ENGINE = SummingMergeTree()
ORDER BY (workspace_id, hour, event_type)
AS SELECT
  workspace_id,
  toStartOfHour(created_at) AS hour,
  event_type,
  count() AS events,
  uniqExact(session_id) AS sessions,
  uniqExact(user_id) AS users,
  sum(revenue) AS revenue
FROM analytics_events_ch
GROUP BY workspace_id, hour, event_type;
```

## Key Architectural Notes

1. **React + Vite for dashboard, Astro for marketing site, Express SSR for landing pages (Option C)** ‚Äî Dashboard is login-only and needs no SEO; public marketing site needs SEO (Astro SSG); One-Click Capture landing pages need dynamic SSR (Express). Three small focused frontends instead of one Next.js mega-app. See [frontend-decision.md](frontend-decision.md).
2. **Shared MySQL/PostgreSQL with `mkt_` prefix** ‚Äî No new DB instance needed in Phase 1‚Äì3. Separate in Phase 4 if query load demands.
3. **DataForSEO as SEO backbone** ‚Äî Single API covers keyword research + SERP + backlinks + technical metrics, replacing the need for separate SEMrush + Ahrefs + Moz subscriptions.
4. **All Bull queues prefixed `mkt-`** ‚Äî Safely coexist on shared Redis with existing LicensedTaxi queues (booking, soc service queues remain untouched).
5. **SSE for One-Click Capture** ‚Äî Server-Sent Events stream progress live. User sees each AI step complete in real-time. No 30-second blank spinner.
6. **AES encryption for all OAuth tokens** ‚Äî Same `encryptPayload/decryptPayload` from existing `encryptionMiddleWare.js`. Social account tokens never stored in plaintext.
7. **Same JWT secret = SSO** ‚Äî Existing LicensedTaxi admin JWT works in marketing platform. One login, both platforms.
8. **React Flow for workflow builder** ‚Äî Drag-and-drop automation canvas. Nodes = triggers, conditions, actions. Edges = flow paths.
9. **Soft deletes everywhere** ‚Äî `deleted_at` on every table. Never `DELETE FROM`. Matches existing platform convention exactly.
10. **Whisper AI for all transcription** ‚Äî Video captions, podcast transcripts, video-to-blog ‚Äî all via OpenAI Whisper (already paying for OpenAI).
